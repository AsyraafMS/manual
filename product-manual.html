eHadir@SMKBN

Staff Attendance & HR Management System — Product Manual

Platform: Android · iOS · Windows · macOS · Linux Stack: Flutter · Supabase · Riverpod Manual version: 1.0 · Generated 2026-05-07

Introduction

eHadir@SMKBN is a Flutter-based staff attendance and HR management application designed for schools. It uses GPS geofencing and Wi-Fi network fingerprinting to verify staff presence, and provides a full workflow for leave management, disciplinary tracking, and real-time admin oversight — all backed by Supabase (PostgreSQL + Auth + Realtime).

Entry point: lib/main.dart — wraps the app in ProviderScope, initialises Supabase from .env, and routes to LoginPage or MainPage via authStateProvider.
Known limitations (pre-production): Default staff passwords are hardcoded (eHadir{Year}!). School/geofence coordinates are partially hardcoded in lib/core/config/. Notification preferences are not yet persisted to the database.

Roles & Permissions

Roles are stored in the profiles table as lowercase strings and parsed via UserRole.fromString() in lib/features/profile/domain/models/profile_model.dart.

Database Tables

All tables are in Supabase PostgreSQL with Row-Level Security (RLS) enabled.

Table Purpose RLS Summary
profiles User profile, role, work hours, notification prefs Own row (staff); all rows (admin)
attendance Daily check-in/check-out records Own rows (staff); all rows (admin)
leaves Leave applications and balance Own rows (staff); all rows (admin)
warnings Disciplinary warning letters Own rows (staff); all rows (admin)
geofences Geofence boundary definition Read: all; Write: system admin
special_holidays Holiday date ranges Read: all; Write: admin
network_fingerprints Trusted Wi-Fi SSID/BSSID pairs Read: all; Write: system admin
notifications In-app notifications Own rows (staff); all rows (admin)
device_logins Device login audit trail Own rows (staff); all rows (system admin)
manual_checkin_requests Manual check-in requests Own rows (staff); all rows (admin)
outstation_requests Outstation/out-of-office requests Own rows (staff); all rows (admin)
exception_requests Attendance exception requests Own rows (staff); all rows (admin)
password_reset_requests Password reset approval queue Own row (staff); all rows (system admin)
notify_prefs Per-user notification preferences Own row only
notify_prefs_defaults School-wide default notification preferences Read: all; Write: admin

Supabase Edge Functions

Server-side functions invoked from the Flutter app for operations requiring elevated privileges.

Function Purpose Caller
admin-reset-password Generate and send a password reset link to a staff member System admin (Password Reset Requests page)
admin-remove-staff Deactivate / delete a staff account from Supabase Auth Admin
admin-set-staff-status Set a staff member's active/inactive status Admin
send-push-notification Deliver FCM push notification to device(s) Internal (triggered by various workflows)
send-reminders Scheduled clock-in and checkout reminder notifications Cron / scheduled trigger
Edge functions use the Supabase service-role key server-side, keeping it off the device. The Flutter app calls them via the Supabase Functions client using the anon key.
